Basics of the Web and HTML

The Worldwide Web

  • What makes up the entire worldwide web is a collection of HTML documents linked together and linked to other files such as videos, text, images, and music
  • HTML (HyperText Markup Language) is the glue that holds the web together
  • Connections between pages are called HYPERLINKS which is what makes the web, weblike

Structure of the Web

There are 4 major components to the web as we know it today.
  1. Browsers make the web viewable from a personal computer and is that space where users experience the web
  2. HTTP (HyperText Transfer Protocol) is the main protocol of the web which allows for the exchange or transfer of hypertext via hyperlinks
  3. The Internet (Cloud) is the world's largest computer network
  4. Servers are stacked computers that host files and html documents that make up the web

What makes up HTML?

There are 4 different components of HTML.
  1. Text Content (what you see)
  2. Markup (what it looks like)
  3. References to other documents e.g. videos, images
  4. Links to other pages

What is HTML Markup?

Html Markup is made up of ELEMENTS that typically contain three parts listed below.

  • Opening Tag which is listed inside brackets sideways carrot brackets like- < >
  • Contents are in between tags
  • Closing Tag is listed inside brackets with an additional slash inside like- </ >

Examples of Tags (note: some tags are "void" tags and don't need a closing tag)

  • b - bold
  • em - emphasis (italics)
  • a - anchor (for making links)
  • img - image, attribute is src - source (url) and alt - text displays when the server fails to load the page
  • br - line break (void tag - no closing tag)
  • p - paragraph

Element Attributes

Attributes are a property of an Element that modifys it e.g. href, src, title, etc. Attributes are always specified in the start tag and come in name/value pairs like: name = "value"

Inline vs. Block Elements

  • Inline Element - functions in line with the text - e.g. span, strong, img, a, br
  • Block Element - actually creates a invisible box that can be modified heighth and width - e.g. div, p, form

Making an HTML Structured Document

House Analogy

HTML is the structure of a website, while CSS is the interior decorations and Java Script is the mechanical parts like a garage door.

"Boxifying" is Key

Everything in HTML can be divided into boxes that can be nested inside of other boxes which allows us to create the structure needed. When a front end web developer gets a design for a webpage from a designer the first step is to identify and outline all the boxes on the page, this is what is meant by "Boxifying."

Need Feedback? - Use a Text Editor

Text editors (e.g. Sublime Text) help programmers by doing certain things automatically like displaying exactly what the code that is written looks like and/or gererating closing tags automatically.

Need More Feedback? - Use Developer Tools

Developer Tools deliver immediate visual feedback about the structure and design of a given page. They also teach us about the nested nature of the code and show us the proper way to indent code we write for ease of reading and editing later on.

How CSS Helps Avoid Repetition

CSS Defined

CSS (Cascading Style Sheets) defines how elements are to be displayed and reduces work by keeping developers from having to repeat adding style code to each individual element in html. CSS code is usually stored in separate files with the file extension .css and is linked in the header of the HTML document.

Cascading Nature of CSS

Inheritance is a key property of CSS to reduce workload and reduce errors when editing code. Inheritance relys on the document structure where parent elements pass on certain traits to child elements such as font, color, or letter spacing. In general only text-related properties can be inherited as opposed to box-related properties (e.g. padding, or margin) This concept helps to avoid errors from repeating steps and also creates consistency in the style of the webpage.

CSS Identifiers

Identifiers allow for us to specify which HTML elements we want to modify and again help us to make consistency in the style of the page and to reduce errors when writing and editing code. One example of this is if we want to reference an element from our HTML we specify the element and inside brackets we can change properties of the element like this: h1{...inside here we adjust properties of this element...} Inside the brackets the change to a property of the element is specified like: name: value; (e.g. color: red;)
